# Vue

- 沿用项目现有路由、数据获取、Store 和目录方案；普通任务不安装新的框架辅助库。
- 远端数据使用项目已有机制，不无理由复制到 Pinia 或其他全局 Store。
- 局部状态留在组件或 composable；只有真实跨边界共享时才提升状态层级。
- 组件在复用、独立测试或降低明显认知复杂度时拆分，不为形式创建单调用者抽象。
- Template 中的组件名遵循项目约定；没有约定时使用 kebab-case，导入符号使用 PascalCase。
- 文件命名和组织沿用现有项目；没有约定时使用通用默认。

复杂响应式、性能或组件设计问题且当前环境已经提供时，可按需使用 `vue-best-practices`。
