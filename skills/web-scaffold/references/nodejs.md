# Node.js

## 既有项目与新项目

- 既有项目沿用当前框架、Schema/校验库、契约、响应结构、目录和错误模型；局部任务不得借机迁移。
- 新建 HTTP 服务且没有既定选择时，优先使用能与框架和契约共源的 Schema 方案，并提供可消费的 OpenAPI。
- 不强制 Zod、TypeBox 或特定分层；避免在 Controller、Route Handler 中重复手写校验。
- 复杂 Node.js 架构问题且当前环境已经提供时，可按需使用 `nodejs-backend-patterns`。

## 执行边界

- 一次只解决一个可独立验收的目标，不用文件数或 Diff 行数衡量风险。
- 超出已确认的模块、契约、数据或依赖边界时停止并重新确认。
- 高风险或跨边界任务先只读分析，给出最小方案、测试、兼容性和恢复路径，确认后再实现。
- 权限、状态机、事务、幂等、并发、缓存、消息、Webhook、数据迁移和外部系统只检查本次实际命中的风险。

## 契约

- HTTP、事件或其他系统边界使用项目现有契约来源；只同步本次受影响的字段、错误码和语义。
- 请求、响应和错误保持显式类型，并验证现有调用方兼容性。
- 生成客户端或模型由契约派生，不手工维护平行类型。
- 联调文档、Swagger UI、CORS、鉴权或代理配置只在本次改变或阻塞验收时处理。

## 测试

TDD 流程与工作区保护以 `references/universal.md` 为唯一规则，本文件不重复。

按本次风险选择测试：

- 业务规则：权限、状态、计算、幂等。
- 契约行为：状态码、字段、错误码和兼容性。
- 集成边界：数据库事务、缓存、消息和外部失败。

优先验证真实业务和持久化结果；Mock 只用于不可控边界，不用调用次数替代行为断言。

## 交付

- 运行受影响测试和项目已有回归、类型检查、Lint、构建。
- 报告契约变化、实际命中的专项风险和剩余风险。
- 涉及外部或不可逆状态时，提供经过确认的恢复或回滚路径。
