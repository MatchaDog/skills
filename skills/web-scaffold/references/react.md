# React

- 沿用项目现有路由、数据获取、状态和目录方案；普通任务不安装新的框架辅助库。
- 远端数据使用现有 Loader、Server Function、生成 Hook 或缓存层，不复制到全局 Store。
- 局部 UI 状态优先使用组件或 URL 状态；只有真实跨边界共享时才提升状态层级。
- `useEffect` 只处理外部系统同步；派生值、事件响应和数据加载优先使用直接表达方式或项目既有机制。
- 组件在复用、独立测试或降低明显认知复杂度时拆分，不为“纯组件”形式拆出单调用者文件。
- 文件命名和组织沿用现有项目；没有约定时使用通用默认。

复杂性能、渲染或组件设计问题且当前环境已经提供时，可按需使用 `vercel-react-best-practices`。
